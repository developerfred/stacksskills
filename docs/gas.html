Language

Clarity

The decidable smart contract language that makes Stacks secure by design.

What is Clarity?

Clarity is a LISP-like smart contract language designed for predictability and security. Unlike Solidity, Clarity is decidable — you can mathematically prove what your code will do before running it.

✅ Why Clarity Matters

No runtime surprises. No hidden behavior. What you write is what executes.

Hello World

A simple Clarity contract looks like this:

;; A simple counter contract
;; Define a variable to store the count
(define-data-map count int u0)

;; Public function to increment
(define-public (increment)
  (let
    ((current (map-get? count "counter")))
    (map-set count "counter" (+ current u1))
    (ok current)
  )
)

Key Differences from Solidity

Feature Solidity Clarity
Recursion Allowed (dangerous) Not allowed (safe)
Type system Static + implicit Static + explicit
Reentrancy Possible (need guards) Impossible (by design)
Syntax C-like LISP-like (S-expressions)

Basic Syntax

Variables

;; Define a constant
(define-constant MAX_VALUE u100)

;; Define a variable
(define-data-map balances principal uint)

Functions

;; Public function (can be called from outside)
(define-public (transfer (amount uint) (recipient principal))
  (begin
    (map-set balances recipient amount)
    (ok true)
  )
)

;; Read-only function (free, no state change)
(define-read-only (get-balance (account principal))
  (default-to u0 (map-get? balances account))
)

Data Types

;; Clarity basic types
u42         ;; unsigned integer (uint)
"hello"     ;; string (buff)
SP1...       ;; principal (Stacks address)
true/false   ;; boolean

Common Patterns

❌ Solidity Style

function transfer(uint amount) public { ... }

✅ Clarity Style

(define-public (transfer (amount uint)) ... )

❌ Solidity Return

return value;

✅ Clarity Return

(ok value) or (err value)
