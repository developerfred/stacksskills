Ship Your dApp

End-to-end guide to building and deploying your first Stacks dApp in 4 steps.

📋 Prerequisites

Before starting, make sure you have:

  • Clarinet installed (brew install clarinet)
  • Hiro Wallet installed
  • VS Code or your favorite editor
1

Initialize Project

Create a new Clarinet project with your first contract.

clarinet new my-first-dapp cd my-first-dapp clarinet contract new counter
2

Write Your Contract

Edit the counter contract to add increment functionality.

;; Increment counter (define-public (increment) (let ((current (var-get count))) (var-set count (+ current u1)) (ok current) ) )
3

Test Locally

Start the dev environment and test your contract.

clarinet devnet start # In another terminal: clarinet console
4

Deploy to Mainnet

When ready, deploy to the real network.

clarinet deploy --network mainnet --contract counter

Key Principles

  • Start small — Most dApps need 1-2 contracts, not 10
  • Test thoroughly — Use Clarinet DevNet before mainnet
  • Keep it simple — Clarity is for ownership, not databases
💰 Cost Check

Stacks mainnet transactions cost fractions of a cent. You can deploy directly without worrying about high fees!
