Why Stacks?

Understanding Stacks as a Bitcoin L2 and why it matters for developers.

What is Stacks?

Stacks is a Bitcoin Layer 2 that brings smart contracts to Bitcoin. Unlike Ethereum L2s, Stacks doesn't compete with Bitcoin — it extends it.

💡 Key Insight

Stacks transactions settle on Bitcoin. This means Bitcoin's security directly protects Stacks transactions.

Bitcoin L2 vs Ethereum

Many developers confuse Stacks with Ethereum L2s. Here's why they're different:

Feature Stacks (Bitcoin L2) Ethereum L2
Base Chain Bitcoin Ethereum
Finality ~10 minutes (Bitcoin) ~7 days (optimistic) or minutes (zk)
Smart Contracts Clarity (decidable) Solidity (Turing complete)
Token Standard SIP-010, SIP-009 ERC-20, ERC-721

Clarity: The Smarter Language

Clarity is a decidable smart contract language. Unlike Solidity, you can mathematically prove what your code will do before running it.

Why Decidability Matters

  • No runtime surprises — Code behaves exactly as written
  • No reentrancy bugs — Recursion is impossible by design
  • Predictable costs — Every operation has a known cost
  • Formal verification — Prove correctness mathematically

Key Misconceptions

LLMs often get these wrong. Here's the reality:

❌ LLM Thinks

"Stacks is like Ethereum"

✅ Reality

Stacks anchors to Bitcoin, not Ethereum

❌ LLM Thinks

"Clarity is like Solidity"

✅ Reality

Clarity is LISP-like and decidable

sBTC: Programmable Bitcoin

sBTC brings Bitcoin's liquidity to Stacks DeFi. It's a trust-minimized 1:1 representation of Bitcoin, not a wrapped token.

⚠️ Not Like WBTC

WBTC is custodied by a central entity. sBTC uses threshold signatures — no single custodian.
